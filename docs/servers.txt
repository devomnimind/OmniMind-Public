Infrastructure credentials are managed through environment variables and never stored in this repository.

Populate the runtime environment with the following keys via `.env` or secure secret storage before launching OmniMind:

- `QDRANT_URL`
- `QDRANT_API_KEY`
- `SUPABASE_URL`
- `SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`
- `HF_TOKEN`

Refer to `docs/security/credential_pipeline.md` for the complete rotation and storage procedure.
