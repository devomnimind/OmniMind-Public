Infrastructure credentials are managed via environment variables and never stored in this repository.

Populate the runtime environment with the following keys via `.env` or the approved secret manager before launching OmniMind:

- `QDRANT_URL`
- `QDRANT_API_KEY`
- `SUPABASE_URL`
- `SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`
- `HF_TOKEN`

Each developer must provision their own secrets and follow the procedures defined in `docs/Modulo Securityforensis/securityAgent.md`.
